# Spec Catalogue Modèles — Yombal.sn

## Objectif

Permettre au client de s'inspirer et de choisir un modèle de tenue à faire confectionner, ou de valider son propre choix.

## Sections attendues

### 1. Header de page
- Titre : "Nos modèles"
- Sous-titre : "Choisissez votre modèle, notre tailleur s'occupe du reste"
- Barre de recherche

### 2. Filtres
- Catégorie (Femme, Homme, Enfant)
- Style (Boubou, Robe, Costume, Kaftan, Tenue de cérémonie, Casual, etc.)
- Occasion (Mariage, Baptême, Quotidien, Soirée, etc.)
- Tailleur associé (optionnel)
- Sur mobile : tiroir de filtres

### 3. Galerie de résultats
- Format galerie (photos larges), 2 colonnes mobile, 3-4 desktop
- Chaque carte modèle :
  - Photo du modèle (belle mise en valeur)
  - Nom / intitulé du modèle
  - Catégorie (Femme / Homme / Enfant)
  - Style (ex. "Boubou cérémonie")
  - Fourchette de prix de confection (ex. "15 000 – 30 000 FCFA")
  - CTA [Choisir ce modèle]

### 4. Option "Votre propre modèle"
- Bloc visible : "Vous avez votre propre modèle ? Uploadez une photo."
- CTA [Uploader mon modèle]

## CTA principaux

| CTA | Action |
|-----|--------|
| Choisir ce modèle | Lance le flux : modèle → tailleur → mesures |
| Uploader mon modèle | Upload photo + flux confection |

## Données affichées

- Photo, nom, catégorie, style, occasion, fourchette prix confection

## Contraintes mobile

- Galerie photo prioritaire : les modèles se choisissent à l'œil
- Texte minimal sur les cartes (photo + catégorie + CTA)
- Bouton "Uploader mon modèle" toujours accessible

## Points à éviter

- Photos de mauvaise qualité
- Trop de texte sur les cartes (c'est une galerie visuelle)
- Absence de fourchette de prix (le client a besoin de se repérer)
- Flux trop long après sélection du modèle
